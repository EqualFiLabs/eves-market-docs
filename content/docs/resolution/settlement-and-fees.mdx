## Redemption

Settlement is market-type and collateral-profile aware. Orderbook CTF positions redeem through ConditionalTokens payout vectors into the market collateral. Parimutuel positions claim against a finalized pool snapshot in the market collateral.

Before resolution, matching CTF YES and NO balances can merge back into collateral. After resolution, holders follow the winning-side redemption path. Settlement preview helpers (`previewCTFRedemption`, `previewParimutuelPayout`, `previewRedemption`) live on the market settlement facet.

For profile markets, redemption and fee accounting stay collateral-native. An eveUSDC market redeems eveUSDC; an eveUSD-profile market redeems eveUSD. Cross-collateral aggregation is an offchain display concern and should be labeled as estimated value if used.

## Fee configurations

Fees are separated by market surface. Each surface has an independent take rate plus a basis-point split that must sum to exactly 10000. The split no longer has a vault leg. Instead it fans out to the senior pool, resolvers, and EvRisk stakers, with a treasury fallback when a recipient is ineligible.

### Orderbook

Take rate `orderbookEntryFeeBps`, default 100 bps. The split is six-way.

| Leg | Default bps |
| --- | --- |
| maker | 4000 |
| creator | 500 |
| protocol | 1000 |
| seniorPool | 2500 |
| resolver | 500 |
| evRisk | 1500 |

### Spot

Take rate `spotTradeFeeBps`, default 100 bps. The split is five-way and has no creator leg.

| Leg | Default bps |
| --- | --- |
| maker | 4000 |
| protocol | 1500 |
| seniorPool | 2500 |
| resolver | 500 |
| evRisk | 1500 |

### Combo

Take rate `comboTradeFeeBps`, default 100 bps. The split is six-way, like orderbook.

| Leg | Default bps |
| --- | --- |
| maker | 4000 |
| creator | 500 |
| protocol | 1000 |
| seniorPool | 2500 |
| resolver | 500 |
| evRisk | 1500 |

### Parimutuel

Take rate `parimutuelEntryFeeBps`, default 250 bps. The split is five-way.

| Leg | Default bps |
| --- | --- |
| creator | 500 |
| protocol | 5000 |
| seniorPool | 2500 |
| resolver | 500 |
| evRisk | 1500 |

### Parlay

Parlay fees live in a separate config, not in the market config. `seniorPoolFeeBps` plus `feeRecipientBps` must sum to 10000. Defaults route all of the fee recipient share to the treasury and none to the senior pool, with an underwriting fee of 3 eveUSDC.

## Fee routing recipients

Each non-maker, non-creator leg has a specific destination, decided inline at fee time.

| Leg | Destination | Fallback |
| --- | --- | --- |
| maker | fee router accrual, claimable by the maker | n/a |
| creator | fee router accrual, claimable by the creator | n/a |
| protocol | treasury | n/a |
| seniorPool | `SeniorCapitalPool.notifyRevenue` | treasury, when token is not eveUSDC or supply is zero |
| resolver | accrued per active epoch to compliant resolvers | treasury, when no epoch is active |
| evRisk | `EvRiskStakingRewards.notifyReward` to active stakers | treasury, when staking is unset |

Because every leg resolves in the same transaction, a fee event is final from an indexer's perspective and does not depend on a later claim.

## Fee claiming

Makers claim accrued maker fees on the fee router facet, and eligible creators claim creator fees for surfaces that carry a creator leg. Spot books have no creator fee. Resolver rewards are finalized per epoch and claimed through the resolver rewards facet, not the market fee router.
