## Scope

Spot books are ERC20 base and quote books. They are the standalone DEX-style surface and should not be described as prediction combo books.

ERC1155 combo books are created with `createComboBook`, use Gnosis CTF positions as base inventory, and belong in the combinatorial positions flow.

## Book identity

The public `computeBookId` signature accepts creator, asset type, transfer mode, base token, token ID, quote token, tick preset, and salt. For ERC20 books, the current library canonicalizes the ID by ordered base and quote token. Integrations should still call `computeBookId` rather than duplicating hashing logic.

| Input | Guidance |
| --- | --- |
| Base token | ERC20 being sold by ask makers. |
| Quote token | ERC20 paid by takers. |
| Tick preset | Determines the generic spot tick size. |
| Salt | Accepted by the ABI, but current ERC20 book identity is canonical by ordered pair. |

## Fees

Spot uses a spot trade fee, not a spot entry fee. The spot fee split has maker, protocol, and vault components. There is no spot creator fee in the current protocol.

Spot book creation can also charge a separate creation fee routed to treasury.

## Transfer modes

The code supports transfer-mode metadata at the book level. Apps should read `baseTransferMode` from `getBookInfo` and follow the live ABI. Do not infer support for ERC1155 spot books from the transfer mode enum.
