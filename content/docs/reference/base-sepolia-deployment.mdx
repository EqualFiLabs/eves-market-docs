## Current status

:::warning Recorded snapshot, pre-refactor
The addresses below are from the last recorded public Base Sepolia deploy, which predates the collateral and fee refactor. Since then eveETH and the sEveUSD vault and lending stack were removed, the fee model gained seniorPool, resolver, and evRisk legs, the resolver registry was split into four facets, and MarketGroupFacet plus FeeConfigFacet were added. Treat the addresses as a historical snapshot and always read live state from the diamond before integrating. A fresh deploy now yields 52 facets and uses eveUSDC as the default collateral and bond token.
:::

## Current source fee defaults

## Current source deploy inventory

A fresh full-stack deploy now includes eveUSDC as collateral and bond token, SeniorCapitalPool, EveIdentity, EvesPositionManager, ParlayTicketToken, and (when the eveUSD stack is enabled) EveUSD, EveRiskShares, EveUSDPool, EveUSDRouter, and EvRiskStakingRewards. Resolver jury economics are hardcoded at deploy: 100 EVE seat stake, active epoch size 16, committees of 3 then 5, one appeal round, one-hour phase windows, quorum 2.

## Integration notes

Call market functions through the diamond. Treat standalone contracts as explicit token, faucet, senior pool, staking, or router integrations.

eveUSDC is 18 decimals even though its backing USDC is 6 decimals. The legacy EVE vote fallback is removed; resolver jury escalation is the committee finality path, gated by `resolutionMode()`.

Market-side orderbook IDs are deterministic, but book records are materialized lazily. Indexers should ingest `MarketCreated`, store deterministic side book IDs, and enrich market, token, and book state through diamond read functions, treating an unmaterialized book as a normal pre-liquidity state.
