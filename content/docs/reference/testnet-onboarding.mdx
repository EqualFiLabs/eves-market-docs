## Test assets

A full-stack test deploy includes mock USDC, mock EVE, eveUSDC, the optional eveUSD pairing pool when enabled, SeniorCapitalPool, a faucet, and the market diamond. The faucet is for testnet onboarding only and dispenses USDC and EVE.

Users usually need USDC, which they wrap into 18 decimal eveUSDC for default market collateral. Resolution and creator bonds use the configured `bondToken`, which defaults to eveUSDC, so onboarding flows should read `getMarketConfig` before telling a user which asset to fund. Resolvers additionally need EVE, because the resolver seat stake is denominated in EVE, not the bond token.

## Basic flow

- Claim test USDC and EVE from the faucet.
- Approve USDC to eveUSDC.
- Wrap USDC into 18 decimal eveUSDC.
- Approve eveUSDC to the diamond.
- If participating as a resolver, approve EVE to the diamond for the seat stake.
- Approve ERC1155 operators, such as ConditionalTokens and the position manager, as required by the route.
- Create or enter a market.
- Trade, resolve, and claim through the diamond.

There is no longer an eveETH asset, eveETH market, or eveETH bond path. Do not instruct users to wrap WETH into eveETH.

## Single-transaction funding

The trade router lets a user skip pre-wrapping. `buyWithUSDC` and `splitWithUSDC` wrap USDC into eveUSDC inline via the wrapper's privileged mint, so a funded USDC balance is enough to enter a market in one transaction. The matching `sellWithUSDC` path unwraps on the way out.

## Allowance display

Frontends should format very large allowances as `Max` or a compact value. Raw `uint256.max` style numbers are not useful in create market modals.

## Smoke paths

Useful user-facing smoke tests include market creation, market group creation, collateral profile creation, scheduled start gating, orderbook split and merge, ask and bid curve fills, combo market creation and branch manipulation, parimutuel entry preview and buy, resolver epoch rotation and dispute, settlement, fee claims, faucet claim, wrapping, and unwrapping.
