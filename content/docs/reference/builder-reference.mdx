## Read surfaces

| Read | Purpose |
| --- | --- |
| `getMarketInfo` | State, timing, fees, outcome, creator, collateral, and volume. |
| `getMarketMetadata` | Question, category, resolution source, start, expiry, and token data. |
| `getMarketTokenInfo` | Position token type, position token, collateral, condition, and token IDs. |
| `getMarketSideBook` | Deterministic YES or NO side book ID. |
| `isBookMaterialized` | Whether a book storage record exists yet. |
| `getBookInfo` | Book pricing, lifecycle, fee, asset, and tick config. |
| `previewParimutuelEntry` | Entry fee, net collateral, minted shares, epoch, and projected pool totals. |

## Execution rules

Taker fills should pass expected generations or expected commitments. Frontends should preview output, enforce slippage, avoid stale curve fills, and distinguish book fills from market-side convenience routes.

For scheduled markets, user-facing actions should check `tradingStartTime` and `expiryTime` before enabling submit buttons.
